# Rolling Cat

一个给真实猫滚键盘用的产品灵感生成器。

猫的键盘输入不会被解释成运势，而是作为随机种子去撞多组创意词库。每个爪印会映射到一张牌：AI 产业能力、真实业务场景、界面/技术形态或越界创意来源；随后用两个 Agent 处理：一个负责神经质、诗意化地读取 nonsense，另一个负责把 nonsense 收敛成能真实落地的 AI 产品。

## 运行

```bash
python3 -m http.server 4173
```

然后打开：

```text
http://localhost:4173
```

## 演示流程

1. 选择今天想要的产品灵感类型。
2. 点击「开始接收」，进入猫键盘安全模式。
3. 邀请猫自由路过键盘、踩一下、蹭一下、尾巴扫一下。
4. 键盘事件会被拦截，只作为随机种子记录，不会触发刷新、跳转或提交。
5. 点击「猫滚完了，生成产品思路」。
6. 页面会抽取词条，先生成一段诗意解答，再生成产品 brief、目标用户、真实痛点、AI 工作流、MVP 和成功指标。

没有猫时可以点「没有猫，我伪装一下」兜底。

## 当前实现

- 静态单页 App，无需安装依赖。
- 单栏极简交互，参考隔壁 Rolling Cats 的降临台流程。
- 猫键盘安全模式会拦截键盘事件。
- 内置多组创意词库：AI 产业能力、真实业务场景、界面/技术形态、越界创意来源。
- 内置双 Agent 提示词：Poetic Agent 负责荒诞诗意解读，Make Sense Agent 必须面向真实可落地的 AI 产品。
- 可选 AI 服务接口：如果提供 `POST /api/idea`，页面会把两段系统提示词和概念卡片传给服务，并用服务返回的 `poem` / `idea` 覆盖本地双 Agent 原型。Make Sense Agent 的提示词会明确要求它不知道输入来源，不能提到猫、随机、键盘、占卜或本产品本身。
- 最终产品 brief 会隐藏输入来源，只把概念卡片当作普通产品灵感输入。
- 支持复制生成出的产品 brief。
